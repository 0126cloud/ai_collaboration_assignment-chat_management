# PRD: 聊天管理後台系統（Chat Management Backstage）

## 1. 概述

### 1.1 產品願景

提供一個完整的後台管理系統，供管理人員即時監控聊天訊息、管理玩家黑名單、審核暱稱變更、發送系統廣播訊息，以及追蹤所有管理操作紀錄。

### 1.2 目標使用者

- **一般管理員（General Manager）**：負責日常聊天監控與玩家管理
- **高級管理員（Senior Manager）**：負責系統管理、帳號管理與廣播訊息

### 1.3 專案範圍

- 前端介面（管理後台 SPA）
- 後端服務（RESTful API）
- 資料儲存設計（含 Mock Data）
- 基本操作流程展示

## 3. 功能模組

### 3.1 使用者認證（Authentication）

管理員透過帳號密碼登入系統，系統驗證身份並根據角色授予對應權限。

**User Stories**：

- As a 管理員, I want to 使用帳號密碼登入, so that 我可以進入後台執行管理工作
- As a 管理員, I want to 修改自己的密碼, so that 我可以維護帳號安全

**角色限制**：所有角色皆可使用

### 3.2 聊天監控（Chat Monitoring）

即時顯示各聊天室的玩家聊天訊息，管理員可搜尋、刪除訊息，並可直接從此頁面封鎖玩家或重設暱稱。

**User Stories**：

- As a 管理員, I want to 查看各聊天室的即時訊息, so that 我可以監控聊天內容是否違規
- As a 管理員, I want to 刪除違規訊息並封鎖玩家, so that 我可以維護聊天室秩序
- As a 管理員, I want to 搜尋特定訊息, so that 我可以快速定位問題內容

**角色限制**：所有角色皆可使用

### 3.3 玩家黑名單管理（Player Blacklist）

管理員可封鎖或解封玩家，並查看黑名單歷史紀錄。封鎖類型包含：垃圾訊息（spam）、辱罵（abuse）、廣告（advertisement）。支援針對特定聊天室或全域封鎖。

**User Stories**：

- As a 管理員, I want to 封鎖違規玩家並選擇原因, so that 封鎖紀錄有完整依據
- As a 管理員, I want to 指定封鎖特定聊天室或全域封鎖, so that 可以根據違規範圍選擇適當的封鎖範圍
- As a 管理員, I want to 解封玩家, so that 誤判或到期的封鎖可以被撤銷
- As a 管理員, I want to 在嘗試重複封鎖時收到提示, so that 我能知道該玩家已在封鎖名單中

**角色限制**：所有角色皆可使用

### 3.4 IP 封鎖（IP Blocking）

封鎖或解封特定 IP 位址，支援精確 IP（如 `116.62.238.199`）和萬用字元模式（如 `116.62.238.*`），並查看封鎖歷史紀錄。

**User Stories**：

- As a 管理員, I want to 輸入精確 IP 或萬用字元格式封鎖 IP, so that 我可以彈性阻止單一 IP 或整個 IP 段的惡意連線
- As a 管理員, I want to 查看與管理已封鎖的 IP 列表, so that 我可以維護封鎖規則
- As a 管理員, I want to 解封 IP, so that 誤判的封鎖可以被撤銷

**角色限制**：所有角色皆可使用

### 3.5 聊天室管理（Chatroom Management）

查看所有聊天室列表及各聊天室的線上人數。此為唯讀模組。

**User Stories**：

- As a 管理員, I want to 查看聊天室列表與線上人數, so that 我可以掌握各聊天室的活躍狀況

**角色限制**：所有角色皆可使用

### 3.6 操作紀錄（Operation Logs）

記錄並顯示所有管理操作（刪除訊息、封鎖玩家、修改帳號等），作為系統稽核追蹤。此為唯讀模組，紀錄由其他模組的操作自動產生。

**User Stories**：

- As a 管理員, I want to 查看所有操作紀錄, so that 我可以追蹤誰在什麼時間做了什麼操作
- As a 管理員, I want to 篩選特定類型的操作紀錄, so that 我可以快速找到相關紀錄

**角色限制**：所有角色皆可使用

### 3.7 系統廣播（Broadcast Message）

高級管理員可發送廣播訊息至指定聊天室或所有聊天室，可設定顯示時長。新增廣播透過「新增廣播」按鈕觸發 Modal 表單，與其他模組的新增操作一致。

**User Stories**：

- As a 高級管理員, I want to 發送廣播訊息到指定聊天室, so that 我可以向玩家傳達重要公告
- As a 高級管理員, I want to 設定廣播顯示時長, so that 訊息在適當時間後自動消失

**角色限制**：僅 Senior Manager

### 3.8 玩家檢舉審核（Player Report Review）

管理員審核玩家提交的聊天檢舉，可核准或駁回。核准檢舉時，系統自動全域封鎖被檢舉玩家。

**User Stories**：

- As a 管理員, I want to 查看待審核的玩家檢舉, so that 我可以處理玩家間的糾紛
- As a 管理員, I want to 核准檢舉並自動封鎖被檢舉玩家, so that 違規行為能被即時處理，無需手動另行封鎖
- As a 管理員, I want to 駁回不成立的檢舉, so that 無辜玩家不會被誤封鎖
- As a 管理員, I want to 依狀態篩選檢舉紀錄（待審核 / 已核准 / 已駁回）, so that 我可以追蹤所有檢舉的處理進度
- As a 管理員, I want to 依檢舉人、被檢舉玩家或時間範圍搜尋, so that 我可以快速定位特定檢舉案件

**角色限制**：所有角色皆可使用

### 3.9 暱稱審核（Nickname Approval）

玩家申請變更暱稱時，需經管理員審核。管理員可核准或駁回申請；駁回時暱稱自動重設為玩家帳號名稱（預設值）。

**User Stories**：

- As a 管理員, I want to 查看所有待審核的暱稱申請, so that 我可以過濾不當暱稱
- As a 管理員, I want to 核准暱稱變更申請, so that 通過審核的暱稱可以在聊天室中正常顯示
- As a 管理員, I want to 駁回暱稱變更申請並自動重設為帳號名稱, so that 不當暱稱不會出現在聊天室中，且玩家仍有有效暱稱可使用
- As a 管理員, I want to 依玩家帳號、申請暱稱或申請時間範圍搜尋, so that 我可以快速找到特定玩家的審核申請

**角色限制**：所有角色皆可使用

### 3.10 帳號管理（Admin Management）

高級管理員可查看所有管理員帳號列表，並執行建立帳號、修改角色、啟用或停用帳號、重設密碼等操作。帳號管理操作會自動記錄至操作紀錄。

**User Stories**：

- As a 高級管理員, I want to 查看所有管理員帳號列表, so that 我可以掌握目前系統帳號狀況
- As a 高級管理員, I want to 建立新的管理員帳號並選擇角色, so that 我可以授權新成員進入系統
- As a 高級管理員, I want to 修改管理員的角色（一般/高級）, so that 我可以調整管理員的操作權限範圍
- As a 高級管理員, I want to 停用或啟用管理員帳號, so that 我可以管控系統存取權限
- As a 高級管理員, I want to 重設管理員密碼, so that 管理員忘記密碼時可由高級管理員協助重設
- As a 高級管理員, I want to 無法對自己的帳號執行停用、角色修改或密碼重設, so that 避免意外鎖定自己的帳號

**角色限制**：僅 Senior Manager

## UX 行為規範

以下為全系統統一 UX 行為，所有頁面皆需遵守：

1. **表單禁用 Enter 送出**：所有 `<Form>` 元件加上 `onKeyDown` handler，阻止 Enter 觸發 `onFinish`。
2. **所有非搜尋 action 前二次確認**：新增、刪除、修改角色、重設密碼、重設暱稱、封鎖、解封等所有會觸發 API 的操作，送出前需顯示 `Modal.confirm`。
3. **新增操作統一用 Modal 表單**：新增類功能（管理員、廣播）一律以按鈕觸發 Modal，不使用 inline form。
4. **搜尋欄位 Input maxWidth 300**：所有 table 搜尋區域的文字 Input 透過 `createStyles` 的 `filterInput: { maxWidth: 300 }` class 設定，不使用 inline style object。
5. **Table 搜尋只能透過按鈕觸發**：篩選條件更動不自動查詢，只有按下「查詢」按鈕才送出請求。
